Przegląd systemu

System synchronizacji BaseLinker-Ovoko to zaawansowane rozwiązanie do automatycznej synchronizacji produktów i zamówień między platformami e-commerce. System zapewnia dwukierunkową komunikację, umożliwiając efektywne zarządzanie inwentarzem i zamówieniami.

BaseLinker → Ovoko

  • • Synchronizacja produktów
  • • Aktualizacja cen i stanów magazynowych
  • • Automatyczne mapowanie kategorii
  • • Obsługa zdjęć produktów

Ovoko → BaseLinker

  • • Import zamówień
  • • Synchronizacja statusów
  • • Automatyczne tworzenie zamówień
  • • Mapowanie produktów

Instalacja i wdrożenie

Wymagania systemowe

  • Node.js wersja 16.0 lub nowsza
  • npm lub yarn do zarządzania pakietami
  • Dostęp do internetu dla API BaseLinker i Ovoko
  • Port 3000 dostępny (konfigurowalny)

Instalacja krok po kroku

1. Klonowanie repozytorium

git clone [repository-url]
cd ovoko-baselinker-sync

2. Instalacja zależności

npm install

3. Konfiguracja

# Edytuj dane uwierzytelniające w server.js
# lub utwórz plik .env
cp .env.example .env

4. Uruchomienie

npm start
# lub dla development
npm run dev

Skrypty npm

npm start Uruchamia serwer produkcyjny
npm run dev Uruchamia serwer deweloperski z hot reload
npm run build Kompiluje frontend do produkcji
npm run check-changes Sprawdza zmiany w produktach

Często zadawane pytania

System synchronizuje się automatycznie co 10 minut dla produktów i co 30 minut dla zamówień. Można to skonfigurować w ustawieniach.

Tak, w panelu kontrolnym można uruchomić synchronizację ręcznie dla produktów i zamówień.

Status można sprawdzić w sekcji Overview panelu kontrolnego oraz w logach aktywności.

Sprawdź sekcję Activity Log, zatrzymaj i uruchom ponownie kolejki, lub skontaktuj się z administratorem systemu.

Bezpieczeństwo i najlepsze praktyki

Bezpieczeństwo

Uwierzytelnianie

  • Wszystkie endpointy API wymagają autoryzacji
  • Hasła są szyfrowane za pomocą bcryptjs
  • Sesje wygasają po 24 godzinach

Dane uwierzytelniające

  • Tokeny API są przechowywane w kodzie (zmień w produkcji)
  • Użyj zmiennych środowiskowych (.env) w produkcji
  • Regularnie rotuj tokeny API

Najlepsze praktyki

Wdrożenie

  • Użyj PM2 lub systemd do zarządzania procesami
  • Włącz HTTPS w produkcji
  • Rozważ migrację do bazy danych (PostgreSQL/MongoDB)

Monitorowanie

  • Regularnie sprawdzaj logi błędów
  • Monitoruj wydajność API
  • Twórz kopie zapasowe plików konfiguracyjnych

Konfiguracja produkcji

Zmienne środowiskowe

# .env
PORT=3000
NODE_ENV=production
BASELINKER_TOKEN=your-token
OVOKO_USERNAME=your-username
OVOKO_PASSWORD=your-password
OVOKO_USER_TOKEN=your-token
SESSION_SECRET=your-secret

PM2 Configuration

{
"name": "ovoko-sync",
"script": "server.js",
"instances": 1,
"env": {
"NODE_ENV": "production"
}
}
